# 征信 API 平台 P10：统计报表 + 成本毛利 + 对账 + 邮件 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development 或 executing-plans. Steps use checkbox (`- [ ]`).

**Goal:** 一期报表闭环：成本侧（product_cost + 每条数据源流水生成成本）、日/月账单、成本统计/月成本、毛利分析（收入−成本，机构/产品/数据源三维）、数据源质量报表、日终三方对账（机构流水↔计费流水↔财务流水），批任务（失败可重跑、幂等），报表邮件（mail_flow）。

**Architecture:** 新建 `yudao-module-report`（§12.2 管理面批任务）。数据源：charge_flow/finance_flow（P6b）、org_flow/ds_flow（P7）、product_cost（本期）。批任务按账期聚合写报表表（按账期分区）。对账核三方一致，差异进 reconcile_record。邮件经 yudao 能力发送记 mail_flow。

**决策记录:**
- **成本归属**：`product_cost`（产品/数据源/接口 × 成功/无数据/失败/超时/特定码 × 单价+生效期+审批）落 `apiten_report`；每条 ds_flow 按 product_cost 生成成本（cost_stat）。
- **批任务框架**：XXL-Job（§12.1）；本期先 Spring `@Scheduled`/手动触发端点 + 幂等（账期+维度唯一，upsert/先删后插）跑通，XXL-Job 注册留基建。
- **跨库聚合**：批任务经 billing/flow 的**账期汇总只读 RPC** 拉数（不跨库直连），report 只存聚合结果。
- **质量报表**：查得率/成功率/P95/切换率从 ds_flow 聚合（P95 批任务算分位数）。

**Tech Stack:** Java 21；`@Scheduled`(起步)/XXL-Job；MyBatis-Plus；OpenFeign（report→billing/flow 账期汇总）；yudao 邮件/excel；H2 单测（聚合喂合成流水）。

**Spec:** v1.3 §5.2.4、§6.3.5、§6.5、§6.7.3、§6.9、§8.5、§9.2、§12.2。

## Global Constraints
- 新建 `yudao-module-report`（`apiten_report`、错误码段 `1_027_xxx_xxx`）。报表表：`product_cost`/`daily_bill`/`monthly_bill`/`cost_stat`/`monthly_cost`/`gross_profit_stat`/`ds_quality_stat`/`reconcile_record`/`mail_flow`。DDL `11-report-schema.sql` + H2。
- 批任务幂等：账期+维度唯一，重跑 upsert/先删后插。
- 每任务一 commit；聚合喂合成数据 H2 测；RPC 拉数离线 stub。

## Tasks（骨架）

### Task 1: report 模块 + apiten_report + product_cost CRUD
- `ProductCostDO { Long id; Integer dimension(1产品 2数据源 3接口); Long refId; String resultCondition; BigDecimal cost; LocalDateTime effectiveTime; LocalDateTime expireTime; Integer auditStatus; }` + 修改记录。CRUD 照搬 P4。
- Commit `chore/feat(report): 新建 yudao-module-report + product_cost 成本规则 CRUD`。

### Task 2: 成本记账（ds_flow → cost_stat）
- 批任务：按账期读 ds_flow（flow RPC 汇总）× product_cost 匹配（维度+结果条件+生效期）→ 聚合 cost_stat。
- Test：合成 ds_flow + product_cost → cost_stat 金额正确（H2 + stub flow）。Commit `feat(report): 成本记账 cost_stat`。

### Task 3: 日/月账单 + 月成本
- `daily_bill`（机构×产品×应答码 × 调用量/计费量/金额）、`monthly_bill`、`monthly_cost`。批任务读 charge_flow（billing RPC 汇总）聚合，幂等。
- Test：合成 charge_flow → daily/monthly_bill（H2）。Commit `feat(report): 日/月账单 + 月成本批任务(幂等)`。

### Task 4: 毛利分析 + 数据源质量报表
- `gross_profit_stat`（机构/产品/数据源 × 收入/成本/毛利/毛利率）= 收入 − 成本。`ds_quality_stat`（查得率/成功率/P95/切换率）从 ds_flow 聚合。
- Test：合成收入+成本→毛利率；ds_flow→质量指标（H2）。Commit `feat(report): 毛利分析 + 数据源质量报表`。

### Task 5: 日终三方对账 + reconcile_record
- 对账任务：按机构/产品/账期核 org_flow ↔ charge_flow ↔ finance_flow 三方一致；差异写 reconcile_record（标记+处理状态）。
- Test：三方一致→无差异；缺一条→差异记录（H2 + stub RPC）。Commit `feat(report): 日终三方对账 + reconcile_record`。

### Task 6: 报表查询/导出/邮件 + mail_flow
- 各报表 HTTP 查询 + 导出（yudao excel）；邮件发送记 `mail_flow`（对象/类型/状态/失败/重发）；任务失败可重跑幂等。
- Commit `feat(report): 报表查询/导出/邮件 + mail_flow`。

## DoD

2. 端到端：跑真实调用后触发批任务 → 日账单/成本/毛利/对账 正确；重跑幂等不翻倍；邮件记 mail_flow。
3. 全量构建 SUCCESS。
