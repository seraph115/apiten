## 1. 文档说明

### 1.1 背景与目标

现有旧 API 体系由 CISP（产品封装、机构查询、运营管理、账单成本）与 EDH（外部数据源接入、统一适配、字段映射、结果码归一）两类系统支撑。新 API 系统（下称"平台"）**全新建设、不考虑旧系统数据迁移**，借鉴旧系统成熟业务模型，建设新一代征信数据服务统一运营平台，通过 API 接口对外提供 JSON 格式数据服务。

平台核心价值主张：**数据源可接入、产品可配置、机构可开通、路由可切换、调用可追踪、费用可计算、账单可统计、问题可排查**。

### 1.2 核心能力清单

1. **上游数据源接入**：对接上游数据接口，做请求入参、出参及错误码映射，通过界面配置包装成平台自有接口，无需编码。
2. **路由切换**：服务接口可通过配置实时变更上游数据源，支持基于规则的失败切换链，以及**按数据价格、质量、请求时延、接口配额多因子打分的动态路由**（策略引擎自动决策 + 人工锁定覆盖）。
3. **计费账单**：计费模板、计费应答码、预付费余额扣减/后付费账单、日月账单、成本统计、毛利分析、合并对账。
4. **请求日志流水**：机构流水、数据源流水、计费流水、财务流水，以雪花 ID 流水号贯穿全链路，可审计、可排查。
5. **多种接口对接方式**：
   - 对外：HTTP/JSON 同步查询、异步申请（回调/轮询）、批量文件任务、开放门户与多语言 SDK。
   - 上游：HTTP/HTTPS、Dubbo/RPC、数据库直连、SFTP/文件交换。

### 1.3 术语

| 术语 | 说明 |
|---|---|
| 数据源 | 外部数据供应商或内部数据能力 |
| 数据源接口 | 数据源下的具体可调用接口 |
| 产品 | 对外可售卖、可授权、可计费、可路由的数据服务单元 |
| 产品功能 | 产品内可独立启停/计费的模块（如工商基本信息、股东） |
| 组合产品 | 多个产品/功能按编排策略组合成的对外产品 |
| 机构 | 平台客户（调用方） |
| 切换链 | 主数据源失败后按序尝试的备用数据源链路 |
| 动态选源 | 按价格/质量/时延/配额多因子打分自动选择上游数据源 |
| 应答码 | 供应商原始应答码或平台统一应答码 |
| 流水号 | 雪花 ID 生成的全局唯一请求标识 |

## 2. 产品定位与角色

### 2.1 系统定位

对内管理数据源、产品、机构、路由、计费、流水、报表和参数；对外提供标准 JSON API 服务、统一认证鉴权、统一响应结构、统一错误码和调用审计。

### 2.2 角色与核心诉求

| 角色 | 核心诉求 | 主要使用界面 |
|---|---|---|
| 运营管理人员 | 维护数据源、产品、机构、计费、路由、参数、报表；掌控成本与毛利 | 管理面全部模块 |
| 技术接入人员 | 数据源接口适配配置、应答码映射、API 文档、联调、排障 | 数据源管理、流水管理、监控运维 |
| 财务/结算人员 | 充值、账务调整、账单核对、对账 | 账户管理、统计报表 |
| 机构客户及其账号 | 调用授权产品；查看账户、账单、流水、调用统计 | 对外 API、机构侧查询界面（二期门户） |
| 系统管理员 | 用户、角色、权限、审计 | 系统管理 |

## 3. 总体架构

### 3.1 三平面架构

```
┌─────────────────────────── 管理面（运营后台）───────────────────────────┐
│ 系统管理 │ 数据源管理 │ 产品信息 │ 机构管理 │ 账户管理 │ 路由管理        │
│ 流水管理 │ 参数管理   │ 统计报表 │ 监控告警 │ 审计日志                  │
└──────────────────────────────┬──────────────────────────────────────┘
                        配置发布/缓存刷新（配置版本化下发）
┌──────────────────────────────▼──────────────────────────────────────┐
│                      控制面（策略与规则引擎）                          │
│ 动态路由引擎（价格/质量/时延/配额多因子评分） │ 切换链编排器            │
│ 计费引擎（模板+应答码规则+余额） │ 限流限额引擎 │ 指标采集器            │
└──────────────────────────────┬──────────────────────────────────────┘
                          路由决策 / 计费结果 / 限流判定
┌──────────────────────────────▼──────────────────────────────────────┐
│                         数据面（执行层）                              │
│ API 网关：认证鉴权→参数校验→限流→路由执行→响应组装→流水异步落盘        │
│ 数据源适配层：HTTP/HTTPS │ Dubbo/RPC │ DB直连 │ SFTP/文件 统一Provider │
│ 异步任务层：异步申请队列 │ 批量文件任务 │ 结果回调/轮询                 │
└──────────────────────────────┬──────────────────────────────────────┘
                               ▼
                    上游数据源（供应商 N 家）
```

### 3.2 核心设计决定

1. **配置与执行分离**：管理面所有运行时配置（路由、计费模板、应答码映射、分流权重、选源策略）修改后处于"草稿"态，经"发布"动作生成**带版本号的配置快照**，推送到控制面/数据面本地缓存，秒级生效；调用链路零数据库依赖；支持一键回滚到任意历史版本；历史流水记录当时命中的配置版本号，可追溯。
2. **一条流水贯穿全链路**：网关入口用雪花 ID 生成全局唯一机构流水号；数据源流水（1:N）、计费流水（1:1）、财务流水全部挂接在该流水号下；任何一次调用可通过流水号还原认证结果、路由决策明细、切换链各节点结果、计费结果。
3. **动态路由是控制面一等公民**：指标采集器实时汇聚各数据源的成功率、查得率、P95 时延、剩余配额、单价；路由引擎按可配置权重策略打分选源；运营可人工锁定覆盖；每次决策打分明细留痕、可解释。
4. **对外三种模式 + 门户**：同步查询、异步申请（回调/轮询取结果）、批量文件任务；二期补开放门户与多语言 SDK。

## 4. 数据面设计

### 4.1 对外 API 服务规范

#### 4.1.1 统一地址规范（JSON over HTTPS）

| 模式 | 地址 | 说明 |
|---|---|---|
| 同步查询 | `POST /api/v1/{productCode}/query` | 实时返回结果 |
| 异步申请 | `POST /api/v1/{productCode}/apply` | 返回申请单号 applyNo |
| 异步结果查询 | `POST /api/v1/{productCode}/result` | 按 applyNo 轮询取结果 |
| 异步回调 | 平台 → 机构回调地址推送 | 带签名，机构返回确认；失败按指数退避重推，默认最多 5 次（可配） |
| 批量提交 | `POST /api/v1/batch/submit` | 文件上传或 OSS 地址 |
| 批量状态 | `GET /api/v1/batch/{taskId}/status` | 任务进度 |
| 批量结果 | `GET /api/v1/batch/{taskId}/result` | 结果文件下载地址 |

接口版本化：路径 `v1` 为大版本；同一产品支持多产品版本并存（`productCode` 绑定产品版本），升级不影响存量机构。

#### 4.1.2 认证鉴权

- **AK/SK + HMAC-SHA256 签名**：签名因子 = AK + 时间戳 + 随机数 nonce + 请求体摘要；时间戳窗口 ±5 分钟防重放；nonce 在窗口内去重。
- **IP 白名单**：账号级配置，支持 IP 段。
- **三级状态校验**：账号启用 → 机构启用 → 机构产品开通且在有效期。
- **密钥轮换**：支持新旧密钥重叠有效期（可配，默认 7 天），到期旧密钥自动作废；历史密钥不可恢复。

请求公共头：

| Header | 必填 | 说明 |
|---|---|---|
| X-App-Key | 是 | 账号 AK |
| X-Timestamp | 是 | 毫秒时间戳 |
| X-Nonce | 是 | 随机数（窗口内唯一） |
| X-Signature | 是 | HMAC-SHA256 签名值 |
| X-Biz-Seq-No | 否 | 机构侧业务幂等键 |
| X-Auth-No | 条件 | 查询授权书编号（个人征信类产品必填） |

#### 4.1.3 统一响应结构

```json
{
  "flowNo": "1943026717538291712",
  "productCode": "P1001001",
  "code": "0000",
  "msg": "成功",
  "charged": true,
  "costTime": 312,
  "data": { }
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| flowNo | string | 雪花 ID 机构流水号（字符串避免 JS 精度丢失） |
| productCode | string | 产品编码 |
| code | string | 平台统一应答码（四段式） |
| msg | string | 应答描述 |
| charged | boolean | 本次是否计费 |
| costTime | number | 平台侧耗时（ms） |
| data | object/null | 产品出参 JSON；无数据/失败时为 null |

#### 4.1.4 平台统一错误码体系（四段式）

| 段 | 含义 | 代表码 |
|---|---|---|
| 0xxx | 成功/无数据 | 0000 成功；0001 查无数据 |
| 1xxx | 认证与权限 | 1001 签名错误；1002 时间戳超窗；1003 IP 不在白名单；1004 账号停用；1005 机构停用；1006 产品未授权；1007 产品已停用/过期；1008 授权书编号缺失 |
| 2xxx | 请求与限制 | 2001 参数缺失；2002 参数格式错误；2003 枚举/长度不合法；2101 余额不足；2102 并发超限；2103 日调用量超限；2104 月调用量超限；2105 幂等冲突（同 bizSeqNo 不同参数） |
| 3xxx | 上游与系统 | 3001 数据源异常；3002 数据源超时；3003 切换链耗尽；3004 数据源限流；3999 系统异常 |

是否计费由"计费应答码"配置决定；默认口径：成功计费、无数据可配、失败不计费。

#### 4.1.5 雪花 ID 方案

- 64 位 = 1 符号位 + 41 位毫秒时间戳 + 10 位工作节点（机房 3 位 + 节点 7 位）+ 12 位序列号；单节点每毫秒 4096 个。
- 网关生成机构流水号；数据源流水号、计费流水号同源生成，通过外键挂接机构流水号。
- 时钟回拨保护：回拨 ≤ 阈值（默认 10ms）自旋等待；超阈值拒绝生成并告警；节点启动时校验上次时间戳。
- 节点 ID 分配：启动时从配置中心/DB 租约获取，防止重复。
- 对外一律以字符串输出。

#### 4.1.6 幂等控制

- 机构可传 `X-Biz-Seq-No`；唯一键 = 机构 + 产品 + bizSeqNo。
- 幂等窗口默认 24 小时（可配）；窗口内重复请求返回首次结果（相同 flowNo、相同 data），不重复计费。
- 同 bizSeqNo 但请求参数摘要不一致 → 返回 2105 幂等冲突。
- 未传 bizSeqNo 的请求不做业务幂等（每次都是新调用）。

#### 4.1.7 参数校验与脱敏

- 按产品入参定义做必填、格式（正则）、枚举、长度校验；校验规则随产品配置下发，网关统一执行。
- 响应敏感字段按产品出参定义的脱敏规则处理（如手机号中间四位、证件号出生段）。
- 日志脱敏：完整 SK、证件号、手机号、银行卡号不得明文出现在任何日志。
- 支持报文级加密（可按机构产品配置开启，AES/SM4 对称 + RSA/SM2 密钥交换）。

#### 4.1.8 异步与批量模式（二期）

- **异步申请**：apply 落队列即返回 applyNo；数据面异步执行完整调用链；结果就绪后回调推送（带签名，指数退避重推最多 5 次）或机构轮询；结果保留期可配（默认 7 天）。
- **批量任务**：机构上传请求文件（CSV/Excel，模板由产品定义）→ 平台逐行执行（复用同步调用链、逐行计费与流水）→ 生成结果文件 → 通知下载；任务级进度、成功/失败行数统计、失败明细文件。

### 4.2 数据源适配服务（统一 Provider）

#### 4.2.1 接入协议

| 协议 | 期次 | 扩展参数 |
|---|---|---|
| HTTP/HTTPS | 一期 | URL、Method、Header 模板、Body 模板、Content-Type |
| Dubbo/RPC | 二期 | 注册中心地址、接口全名、方法、版本、分组、序列化 |
| 数据库直连 | 二期 | 连接串（加密存储）、SQL 模板、连接池参数 |
| SFTP/文件交换 | 二期 | 主机、目录、文件名模板、编码、解析模板、轮询频率 |

Provider 接口抽象一期就绪：`request(标准入参, 接口配置) → 标准出参`，新协议以插件方式扩展，不影响既有产品。

#### 4.2.2 配置化适配（界面配置，无需编码）

1. **请求模板**：URL/Header/Body 模板 + `${param}` 变量占位。
2. **入参映射**：平台标准参数 → 供应商参数；支持转换函数库：加密（MD5/SHA/AES/SM 系列）、日期格式转换、证件类型码表转换、拼接/截取、默认值。
3. **出参映射**：JSONPath/XPath 抽取 → 平台标准字段；支持数组展开、字段重命名、类型转换、空值策略。
4. **应答码映射**：供应商原始码 → 平台统一码，附带四个标记：是否成功、是否计费、是否可重试、是否触发切换。未映射的原始码归一为 3001 并告警提示补录。
5. **联调测试台**：界面上输入测试参数 → 实际调用上游 → 展示原始请求/原始响应/映射后结果，联调通过才允许启用接口。

#### 4.2.3 供应商侧安全与容错

- 认证方式：Token（含获取/刷新流程配置）、签名、证书（双向 TLS）、IP 白名单出口（固定代理出口 IP）。
- 报文加解密：AES/RSA/SM2/SM4，密钥加密存储。
- 超时：连接超时与读取超时分开配置；重试：次数 + 退避策略（固定/指数），仅对"是否可重试=是"的失败重试。
- 留痕：每次真实外调记录数据源流水；原始报文摘要加密留存，保留期按合规可配，到期归档/销毁。

#### 4.2.4 健康检查

- 主动探活：接口级可配探测报文与频率，探测结果不计费不入账单。
- 被动统计：滑动窗口（默认 5 分钟）成功率、查得率、P95 时延、切换率。
- 健康状态（健康/亚健康/摘除）供路由引擎与监控看板消费。

## 5. 控制面设计

### 5.1 路由决策模型（三层递进）

#### 第一层：静态路由匹配（谁说了算）

- 优先级：**机构产品路由 > 机构路由 > 产品默认路由**。
- 附加匹配条件：请求参数（如地区码、证件类型）、产品功能、账号、时间段。
- 匹配结果指向一个路由目标，四种类型：
  1. 单数据源接口
  2. 权重分流组（多个数据源按权重比例分流，支持固定优先级）
  3. 切换链
  4. **动态选源策略**

#### 第二层：动态选源（选哪个上游）

当路由目标为"动态选源策略"时，对候选数据源集合实时打分：

```
score = w1·价格分 + w2·质量分 + w3·时延分 + w4·配额分
```

| 因子 | 计算口径 | 数据来源 |
|---|---|---|
| 价格分 | 候选中最低成本=100，线性归一 | 产品成本配置（区分成功/无数据单价） |
| 质量分 | 查得率 × 准确率（准确率默认 1，可人工标定） | 指标采集器滑动窗口 |
| 时延分 | P95 时延，最快=100 线性归一 | 指标采集器滑动窗口 |
| 配额分 | 剩余配额比例（日/月取小） | 配额账本 |

- 权重 w1–w4 由**选源策略模板**配置（和为 100），可按产品/机构覆盖。
- 滑动窗口长度默认 5 分钟，可按策略模板配置。
- **保护机制**：
  - 熔断：成功率低于阈值自动摘除，半开探测恢复；
  - 最小流量保底：每个健康候选源保留最小流量比例（默认 5%），避免指标失真；
  - 人工锁定：运营强制指定源，优先级最高，锁定有生效期与审计；
  - 决策留痕：每次打分明细（各候选分值、命中原因）写入机构流水扩展字段。

#### 第三层：切换链兜底（失败了怎么办）

- 触发条件：应答码"是否触发切换"标记、超时、无数据（可按规则配置）、调用异常、成功率阈值。
- 切换规则：是否继续切换、切换目标、最大切换次数、切换后是否计费、规则优先级（冲突时高优先级生效）、规则试运行（只记录不生效）与启停。
- 切换链：链路顺序、每节点切换条件、终止条件、兜底数据源、返回合并策略、每节点独立计费策略。
- 切换链版本化：历史请求可追溯当时命中的链路版本；流水记录每节点调用结果。

### 5.2 计费引擎

#### 5.2.1 计费判定

调用结束后按 **机构产品 → 计费模板 → 计费应答码** 三级配置判定是否计费、计多少：

1. 取机构产品绑定的计费模板与单价；
2. 按平台应答码查计费应答码规则（机构/产品/模板级覆盖 > 默认）：成功计费、无数据计费与否、失败不计费、特定错误码计费；
3. 按收费模式计算金额。

#### 5.2.2 收费模式

| 模式 | 规则 |
|---|---|
| 按次单价 | 每次计费调用按机构产品单价 |
| 阶梯计费 | 按账期累计计费量落档；**采用滑档口径**：每档内的量按该档单价计（非返算） |
| 免费额度 | 账期内先扣免费额度，超出部分计费 |
| 最低消费 | 账期结算时不足最低消费按最低消费收取 |
| 包时段套餐 | 二期 |

#### 5.2.3 扣费与一致性

- 预付费：实时扣减账户余额；余额不足按机构配置：拒绝调用（2101）或透支至信用额度。
- 后付费：记账单，账期结算。
- 一致性保障：
  - 账户余额扣减乐观锁 + 计费流水幂等（一个机构流水号至多一条计费流水）；
  - 计费流水与余额变动同事务；
  - 已扣费但调用最终失败的场景自动冲正（生成冲正财务流水）；
  - 日终对账任务自动核对 机构流水 ↔ 计费流水 ↔ 财务流水 三方一致性，差异进合并对账。

#### 5.2.4 成本侧

每条数据源流水按产品成本配置（产品/数据源/接口维度 × 成功/无数据/失败/超时/特定应答码）生成成本记录；成本规则带生效期、审批状态与修改记录；支撑毛利分析（收入 − 成本，按机构/产品/数据源三维）。

### 5.3 限流限额引擎

| 能力 | 规则 | 超限行为 |
|---|---|---|
| 并发控制 | 机构/账号/产品三级最大并发（令牌桶/信号量） | 可配：拒绝（2102）/ 排队（最大等待时长）/ 降级 |
| 调用量控制 | 日/月/账期累计上限，机构/账号/产品维度 | 达 80%/95% 预警（邮件+告警）；超限阻断（2103/2104）或转审批 |
| 数据源配额 | 供应商日/月配额账本，每次外调扣减 | 接近耗尽降低路由得分直至摘除；恢复后自动回流 |

并发与限额配置实时生效；提供当前并发水位、历史峰值、触发记录查询。

### 5.4 配置发布机制

- 适用对象：路由配置、选源策略、切换规则/链、分流权重、计费模板、计费应答码、应答码映射、限流限额参数。
- 流程：草稿 → 发布（生成版本快照，敏感对象需审批）→ 推送刷新各节点缓存（秒级）→ 可回滚至任意历史版本。
- 每次发布记录：发布人、时间、变更 diff、影响范围（哪些产品/机构）；流水记录调用时配置版本号。

## 6. 管理面设计（字段级）

> 通用要求：所有管理列表具备分页、条件组合查询、详情、新增、修改、启停；关键操作记录操作日志（操作人、时间、变更前后值）；敏感操作（账务调整、路由发布、成本修改）二次确认，其中账务调整与成本修改需审批流。

### 6.1 系统管理

| 功能 | 字段/要点 |
|---|---|
| 后台用户 | 用户名、姓名、手机号、邮箱、状态、所属角色（多）、最近登录；登录/退出/密码修改/重置/账号启停 |
| 角色 | 角色名、说明、菜单权限（树）、按钮权限、数据权限范围 |
| 菜单 | 名称、路径、图标、排序、父级、按钮点位 |
| 操作日志 | 操作人、时间、模块、动作（增/删/改/启停/导入/导出/审核/发布/回滚）、对象、变更前值、变更后值、IP |
| 登录日志 | 用户、时间、IP、结果、异常登录提示 |
| 参数字典 | 系统参数、字典项、枚举值统一维护 |

### 6.2 数据源管理

#### 6.2.1 数据源信息

字段：数据源编码（唯一，平台生成规则 DS+序号）、名称、供应商名称、数据源类型（供应商/内部）、联系人、联系方式、状态、环境标识（生产/测试）、服务地址、认证方式、超时时间、最大并发、重试次数、是否参与路由、接入协议类型、协议扩展参数、备注。
详情页聚合：下挂接口数、今日调用量、成功率、失败率、最近调用时间、健康状态、配额使用。

#### 6.2.2 数据源接口

字段：接口编码、名称、所属数据源、URI、请求方式、报文格式、签名方式、加密方式、认证参数、请求参数定义（名称/类型/必填/说明）、响应参数定义、接口版本、状态、超时、重试、缓存策略（是否缓存/TTL/缓存键）、成本关联、接口文档（请求示例/响应示例/字段说明）。
操作：新增、修改、查询、详情、启停、**联调测试台**、与产品功能映射管理。

#### 6.2.3 数据源应答码

字段：所属数据源/接口、原始应答码、应答描述、是否成功、是否计费、是否可重试、是否触发切换、映射平台统一码。
操作：按数据源/接口/产品维度查询；批量导入（Excel 模板）/导出；未映射码告警列表。

#### 6.2.4 分流配置

字段：分流编码、名称、适用维度（产品/机构/接口/区域/请求参数）、分流方式（单源/权重/固定优先级）、成员及权重、状态、生效时间、失效时间、版本。
分流命中结果写入机构流水。

#### 6.2.5 切换规则

字段：规则编码、名称、触发条件（错误码集合/超时/无数据/异常/成功率阈值）、是否继续切换、切换目标、最大切换次数、切换后是否计费、优先级、状态、试运行开关。

#### 6.2.6 切换链

字段：链编码、名称、节点列表（顺序、数据源接口、节点切换条件、节点计费策略）、终止条件、兜底数据源、返回合并策略、版本、状态。
详情页展示链路图与每节点应答码规则；版本历史可查。

#### 6.2.7 配额账本

字段：数据源/接口、配额周期（日/月）、配额总量、已用量、剩余量、告警阈值、重置时间。

### 6.3 产品信息管理

#### 6.3.1 产品信息

字段：产品编码（P+分类+序号）、名称、产品类型（企业/个人/核验/司法/经营风险/知识产权/报告/组合）、认证类型、状态、版本、说明、入参定义（含校验规则）、出参定义（含脱敏规则）、示例报文、错误码说明、是否缓存、是否支持异步、是否需要授权书编号。
操作：上下架、启停、版本升级（多版本并存）、历史版本查看。

#### 6.3.2 产品功能

字段：功能编码、名称、所属产品、排序、是否必选、是否计费、状态、映射数据源接口（多）。

#### 6.3.3 组合产品

字段：组合产品编码、名称、组成项（产品/产品功能）、执行顺序、并行/串行策略、结果合并策略、失败处理策略（任一失败即失败/部分成功返回）、成本汇总（自动）、状态。
详情页展示组成产品、数据源、计费规则和路由规则全景。

#### 6.3.4 产品数据源配置

字段：产品/功能 ↔ 数据源接口绑定、默认数据源、备用数据源、分流规则、切换链、**选源策略模板**；支持按机构覆盖。
详情页展示产品到数据源的完整调用路径图。

#### 6.3.5 产品成本管理

字段：维度（产品/数据源/接口）、结果条件（成功/无数据/失败/超时/特定应答码）、成本单价、生效时间、失效时间、审批状态。
产品成本修改记录：修改人、时间、变更前后值、审批人。

### 6.4 机构管理

#### 6.4.1 机构信息

字段：机构编码（ORG+序号）、名称、统一社会信用代码、联系人、联系方式、状态、业务归属、备注。
机构停用即时阻断 API 调用并记录原因；资料变更记录可查。
详情页聚合：账户、产品、账号、计费模板、调用统计。

#### 6.4.2 机构账号

字段：账号、AK、SK（加密存储、界面仅显示一次/重置显示）、密钥状态、IP 白名单、回调地址、签名算法、有效期、账号级并发/调用量限制、状态。
操作：密钥生成、重置、停用、历史密钥作废；密钥轮换（新旧重叠期）。

#### 6.4.3 机构产品管理

字段：机构、产品、状态、有效期、单价、计费模板、调用限额、并发限制。
修改记录：开通、停用、调价、换模板等动作全记录。
**开通向导**：7 步引导（机构 → 账号 → 白名单/签名 → 开通产品 → 计费模板/限额 → 账户充值/授信 → 联调启用）。

#### 6.4.4 计费模板

字段：模板名称、收费模式（按次/阶梯/免费额度/最低消费）、指定单价、阶梯区间表、免费额度、最低消费、结算周期（自然月/账期日）、状态、版本。
操作：复制、新增、修改、启停、版本记录；详情展示适用产品与机构。

#### 6.4.5 计费应答码

字段：平台应答码、是否计费、适用范围（默认/机构/产品/模板覆盖）、说明。

#### 6.4.6 机构用户映射与机构用户管理

机构用户（客户侧人员）、后台用户、业务人员映射；一个机构多个业务用户；增改启停查。

#### 6.4.7 合并对账

按机构、产品、账期、账户维度合并对账；机构调用流水 ↔ 计费流水 ↔ 财务流水差异核对；对账结果查询、导出、差异标记与处理状态。

### 6.5 账户管理

| 功能 | 字段/要点 |
|---|---|
| 计费账户 | 账户编号、机构、账户类型（预付费/后付费）、余额、冻结金额、信用额度、状态；累计充值、累计消费、最近消费时间；启停/冻结/解冻/**余额预警阈值** |
| 账户充值 | 金额、方式、凭证（附件）、经办人、审批状态、备注；成功后生成财务流水并更新余额；记录查询导出 |
| 财务流水 | 类型（充值/消费/退款/调整/冻结/解冻/冲正）、机构、账户、时间、产品、订单号、关联调用流水/账单/操作记录 |
| 账务调整 | 调增/调减/退款/冲正、原因、凭证、审批状态、调整前后余额；完成后生成财务流水与审计日志 |

### 6.6 路由管理

- 通用路由：路由编码、名称、产品、机构、目标（单源/分流组/切换链/选源策略）、附加条件（参数/地区/功能/账号/时间段）、优先级、状态、生效/失效时间、变更记录。
- 三级路由配置视图：产品默认路由、机构专属路由、机构产品路由。
- **选源策略模板**：策略名、w1–w4 权重、滑动窗口、熔断阈值、最小流量比例、人工锁定入口。
- **路由模拟器**：输入机构+产品+参数 → 预演命中的路由/分流/选源打分结果，不真实外调。
- **配置发布**：草稿/发布/回滚，见 §5.4。
- 路由命中结果写入机构流水与数据源流水。

### 6.7 流水管理

#### 6.7.1 机构流水

记录字段：流水号（雪花 ID）、机构、账号、产品、产品功能、bizSeqNo、请求时间、响应时间、耗时、请求状态、平台应答码、是否计费、计费金额、命中路由（含版本）、选源打分明细、切换链（含版本）、请求 IP、签名结果、授权书编号、请求摘要、响应摘要。
查询：机构、产品、应答码、时间、流水号、bizSeqNo；导出。
**全链路视图**：流水号一键展开 认证 → 限流 → 路由决策 → 各数据源节点 → 计费 全过程。

#### 6.7.2 数据源流水

记录字段：数据源流水号、关联机构流水号（1:N）、数据源、接口、供应商请求号、请求/响应时间、耗时、原始应答码、平台应答码、是否切换、切换序号、是否计成本、成本金额。

#### 6.7.3 邮件发送流水

发送对象、类型（账单/报表/告警/通知）、时间、状态、失败原因、重发记录与重发操作。

### 6.8 参数管理

- 机构并发量控制：机构/账号/产品维度最大并发、超限策略（拒绝/排队/降级）、当前并发水位、历史峰值、触发记录。
- 机构调用量控制：日/月/账期上限、预警阈值（80%/95% 默认可改）、超限行为（阻断/审批）、控制记录查询。

### 6.9 统计报表

| 报表 | 维度与指标 |
|---|---|
| 日账单 | 机构、产品、应答码 × 调用量、计费量、金额 |
| 月账单 | 账期 × 机构月度消费汇总与明细 |
| 成本统计 | 数据源、接口、产品、时间 × 供应商成本 |
| 月成本 | 数据源、产品、机构 × 月度成本汇总 |
| 毛利分析 | 机构/产品/数据源三维 × 收入、成本、毛利、毛利率 |
| 数据源质量报表 | 数据源/接口 × 查得率、成功率、P95 时延、切换率（供选源策略调参） |
| 综合指标 | 收入、成本、毛利、调用量、成功率、无数据率、失败率 |

全部支持查询、导出、邮件发送（生成邮件流水）；账单任务失败可重跑、幂等。

### 6.10 监控运维

- API 监控：调用量、成功率、失败率、平均耗时、P95/P99。
- 数据源监控：可用率、超时率、错误码分布、切换次数、配额水位。
- 告警：机构调用异常、余额不足、限额触发、数据源异常、未映射应答码、时钟回拨、配置发布失败。
- 排障：按流水号定位全链路；决策可解释性查询。
- 运维：配置缓存刷新、关键任务调度（日/月账单、成本统计、流水归档、对账）、任务运行记录与重跑。

## 7. 核心数据模型（9 域约 40 表）

```
数据源域: data_source ─< ds_interface ─< ds_response_code
                          │                （原始码→平台码映射+4标记）
                          ├─< ds_interface_param（入/出参映射定义）
                          └─< ds_quota_ledger（配额账本）
产品域:   product ─< product_function >─ ds_interface（功能-接口映射）
          product ─< product_param（入参/出参/校验/脱敏定义）
          product ─< product_cost（成本规则：维度+应答码+生效期+审批）
          combo_product ─< combo_item（组合项+执行策略）
机构域:   org ─< org_account（AK/SK+白名单+回调+限额）
          org ─< org_product（开通+单价+模板+限额+有效期）>─ billing_template
          billing_template ─< template_tier（阶梯区间）
          charge_code_rule（计费应答码：默认/机构/产品/模板覆盖）
          org_user / org_user_mapping
账务域:   billing_account（余额+冻结+信用额度+预警阈值）
          ─< recharge_record ─< finance_flow ─< adjust_record
路由域:   route_config（三级路由+条件+目标+优先级）
          dynamic_strategy（w1-w4+窗口+熔断+保底）
          split_config（分流）/ switch_rule / switch_chain ─< chain_node
          config_release（配置版本快照+发布/回滚记录）
流水域:   org_flow（机构流水，雪花ID主键，含路由/策略/链版本+打分明细）
          ─< ds_flow（1:N 数据源流水）
          ── charge_flow（1:1 计费流水）── finance_flow
          mail_flow / idempotent_record（幂等记录）
控制域:   concurrency_ctrl / volume_ctrl / threshold_alert_record
报表域:   daily_bill / monthly_bill / cost_stat / monthly_cost
          gross_profit_stat / ds_quality_stat / reconcile_record
系统域:   sys_user / sys_role / sys_menu / sys_perm / op_log / login_log / dict
```

关键约束：

- 流水表按月分表 + 冷热分离归档；报表表按账期分区。
- `org_flow` 记录 `route_version`/`strategy_version`/`chain_version`/`template_version`。
- 敏感字段（SK、证件号、手机号、银行卡号）AES/SM4 加密存储。
- 编码规则统一：DS（数据源）、IF（接口）、P（产品）、F（功能）、ORG（机构）、R（路由）、T（模板）前缀 + 序号。

## 8. 关键业务流程

### 8.1 数据源接入流程

1. 新增数据源信息 → 2. 新增数据源接口（URI/认证/参数/响应字段/协议扩展）→ 3. 配置入参/出参映射与转换函数 → 4. 配置应答码映射（含四标记）→ 5. 配置接口成本 → 6. 绑定产品功能 → 7. 配置路由/分流/切换链/选源策略 → 8. 联调测试台验证通过 → 9. 发布启用。

### 8.2 产品上架流程

1. 新增产品（编码/类型/版本）→ 2. 定义入参出参与校验/脱敏规则 → 3. 新增产品功能并绑定数据源接口（或组合已有产品）→ 4. 配置默认路由、成本、错误码 → 5. 维护 API 文档与示例报文 → 6. 启用并允许机构授权。

### 8.3 机构开通流程（开通向导）

1. 新增机构 → 2. 创建账号生成 AK/SK → 3. 配置 IP 白名单/签名/有效期 → 4. 开通机构产品 → 5. 绑定计费模板、调用量/并发控制 → 6. 创建计费账户并充值/授信 → 7. 联调通过后启用。

### 8.4 API 调用流程（同步）

```
① 网关五重校验（签名/IP/账号/机构/产品，纯缓存）
② 前置检查（余额、日/月调用量、并发水位）
③ 幂等检查（bizSeqNo 命中→直接返回首次结果）
④ 雪花ID生成机构流水号
⑤ 路由决策（静态匹配→动态选源打分→确定目标）
⑥ 适配层调用上游（入参映射→调用→出参映射→应答码映射）
⑦ 失败？→ 按切换规则进入切换链下一节点（回⑥）
⑧ 计费判定→余额扣减→计费流水
⑨ 统一响应返回（平台开销目标 P99 < 50ms）
⑩ 异步落盘（机构流水+数据源流水+决策明细，MQ 削峰）
```

### 8.5 计费与对账流程

1. 调用结束按三级配置判定计费 → 2. 预付费扣余额/后付费记账 → 3. 生成计费明细与财务流水 → 4. 日终日账单、月终月账单 → 5. 收入成本核对（机构/产品/数据源）→ 6. 差异走合并对账与账务调整（审批+冲正）。

### 8.6 异步与批量流程（二期）

- 异步：apply 入队 → 异步执行 §8.4 ④–⑧ → 结果就绪 → 回调推送（签名+重推）/轮询 → 结果保留期后清理。
- 批量：文件校验入库 → 逐行执行（逐行流水与计费）→ 结果文件生成 → 通知下载 → 任务级统计。

## 9. 非功能需求

### 9.1 性能

- 网关自身开销 P99 < 50ms；同步链路整体耗时 ≈ 上游耗时 + 50ms。
- 单节点 ≥ 500 TPS，数据面无状态水平扩展至万级 TPS。
- 后台列表查询分页 + 索引优化；流水/报表分表、归档、冷热分离。

### 9.2 可用性

- 数据面多活部署，目标可用性 99.95%。
- 配置中心/数据库故障时，数据面凭本地缓存快照降级运行（只影响配置变更，不影响调用）。
- 数据源异常：超时、重试、熔断、切换链、兜底响应。
- 账单、统计、报表任务失败重跑且幂等。

### 9.3 安全与合规（征信场景）

- API 调用完成账号、签名、IP、产品权限校验；防重放。
- 个人征信类产品强制授权书编号留痕。
- 密钥、证件号、手机号、银行卡号加密存储或脱敏展示；日志全面脱敏。
- 原始报文留存与销毁周期可配，满足合规审计。

### 9.4 审计

- 产品、机构、计费、路由、数据源、成本等关键配置变更全审计（操作人/时间/前后值）。
- 账务调整、充值、退款、冲正记录操作人、审批状态、凭证。
- 任一调用可通过流水号追溯机构、产品、路由（含版本与打分明细）、数据源、计费结果。

### 9.5 可扩展性

- 新增数据源不影响既有产品；新增产品优先配置组合已有接口。
- 上游协议以 Provider 插件扩展；对外模式（异步/批量/门户）在网关分层上叠加。
- 产品多版本并存，升级不影响存量机构。

### 9.6 数据一致性

- 计费流水与余额变动同事务；计费流水按机构流水号幂等。
- 日终三方对账（机构流水 ↔ 计费流水 ↔ 财务流水），差异进合并对账。

## 10. 分期建设

### 10.1 一期（运营闭环）

- 系统管理：用户、角色、菜单、权限、操作日志。
- 数据源管理：数据源、接口（HTTP 适配 + 联调测试台）、应答码映射、配额账本。
- 产品信息：产品、功能、产品数据源、成本（含修改记录）。
- 机构管理：机构、账号、机构产品（含开通向导与修改记录）、计费模板、计费应答码。
- 账户管理：计费账户、充值、财务流水、账务调整。
- 路由管理：三级静态路由、分流、切换规则、切换链、配置发布/回滚、路由模拟器。
- 流水管理：机构流水、数据源流水、全链路视图；邮件流水（视资源）。
- 参数管理：并发控制、调用量控制。
- 统计报表：日/月账单、成本统计、月成本、毛利分析。
- 对外 API：同步查询、认证鉴权、统一响应、错误码、幂等、雪花 ID、计费。
- 监控：基础看板与告警。
- **动态选源的指标采集一期即开始**（积累数据供二期策略调参）。

### 10.2 二期（智能与开放）

- 动态选源策略引擎自动决策（含熔断、保底、人工锁定、决策留痕）。
- 异步申请 + 回调/轮询、批量文件任务。
- 开放平台门户、在线调试、多语言 SDK、API 文档站。
- 上游 RPC/DB/文件适配插件。
- 合并对账自动化、供应商 SLA 分析、数据质量看板。
- 图谱分析、股权穿透、最终受益人、PDF/Excel 报告生成。
- 数据源低代码适配增强。

> 说明：范围划分继承需求文档 §10 并按三平面架构重排；动态选源自动决策放二期是因为需要一期积累的真实指标数据才能调出可用策略，一期先提供指标可视与人工切换。

## 11. 验收标准

继承需求文档 §11 全部 8 条：

1. 后台可完成数据源、接口、应答码、产品、机构、账号、路由、计费模板完整配置。
2. 机构账号 AK/SK 调用授权产品成功；未授权产品返回统一错误码。
3. 调用按路由配置命中指定数据源；数据源异常按切换规则进入备用链路。
4. 每次调用生成机构流水与数据源流水，流水号可查完整链路。
5. 成功调用按计费模板生成计费结果；余额、财务流水、日/月账单数据一致。
6. 并发量、调用量达阈值可拦截或预警。
7. 关键配置变更有操作审计。
8. 报表按机构、产品、数据源、时间维度查询导出。

追加 4 条：

9. 动态选源策略下，调整权重后新请求按新打分选源，流水可查打分明细。
10. 数据源配额耗尽自动降权直至摘除，配额恢复后自动回流。
11. 同一 bizSeqNo 重复请求返回相同流水号且只计费一次。
12. 任一运行时配置可回滚到历史版本，回滚有审计记录。

## 12. 技术架构

> 全新建设，不考虑旧系统迁移。

### 12.1 总体选型

- **基座框架**：**yudao-cloud**（芋道，ruoyi-vue-pro 的 Spring Cloud Alibaba 微服务版，https://github.com/YunaiV/yudao-cloud）。直接复用其系统管理（用户/角色/菜单/按钮权限/操作日志/登录日志/字典/参数配置，即 PRD §6.1 全部能力）、网关、MQ 抽象、代码生成、监控接入等基础设施；业务服务按 yudao 模块规范新建。
- **后端**：Java 21（虚拟线程）+ Spring Boot 3.x + Spring Cloud Alibaba（Nacos/Sentinel/OpenFeign/Gateway）+ MyBatis-Plus + Redisson。
- **前端**：yudao-ui-admin-vue3（Vue 3 + TypeScript + Vite + Element Plus），复用其 RBAC 动态菜单、字典、CRUD 配套页面。
- **数据与中间件**：MySQL 8、Redis、**Kafka**（经 yudao MQ 抽象层接入，流水削峰/异步任务/回调重推）、Nacos、Sentinel、ShardingSphere-JDBC（流水按月分表，自行引入）、XXL-Job、SkyWalking、Prometheus/Grafana。

### 12.2 微服务划分（yudao-cloud 模块规范）

| 服务/模块 | 平面 | 职责 | 关键技术 |
|---|---|---|---|
| yudao-gateway（扩展） | 数据面 | 对外统一入口：在 yudao 网关上扩展签名/IP/防重放校验过滤器、Sentinel 限流、雪花 ID 生成、幂等检查 | Spring Cloud Gateway |
| yudao-module-openapi | 数据面 | 调用编排：参数校验、路由决策（内嵌引擎 + 本地配置快照）、切换链执行、计费触发、统一响应组装 | 虚拟线程、OpenFeign |
| yudao-module-adapter | 数据面 | 统一 Provider：入参/出参/应答码映射执行、供应商调用（HTTP 一期，RPC/DB/文件插件二期）、健康探活 | 虚拟线程、按数据源连接池隔离 |
| yudao-module-route | 控制面 | 路由/分流/切换链/选源策略配置管理、指标聚合与选源打分参数计算、配置快照发布与版本回滚 | Nacos 配置发布、Redis Pub/Sub 缓存刷新 |
| yudao-module-billing | 控制面 | 计费判定、余额扣减（本地强事务 + 乐观锁）、计费流水、账户/充值/财务流水/账务调整 | MySQL 本地事务、冲正补偿 |
| yudao-module-flow | 数据面 | 流水异步消费落库、机构/数据源流水查询、全链路视图 | Kafka 消费、ShardingSphere 按月分表 |
| yudao-module-system（复用） | 管理面 | 系统管理全部能力：用户、角色、菜单、按钮权限、操作日志、登录日志、字典 | yudao 自带，零开发 |
| yudao-module-apiadmin | 管理面 | 业务管理后台：数据源/产品/机构/账户/参数配置 CRUD、审计、报表查询 | MyBatis-Plus + yudao CRUD 脚手架/代码生成 |
| yudao-module-report | 管理面 | 日/月账单、成本统计、对账、归档、邮件报表批任务 | XXL-Job，幂等可重跑 |

调用关系（同步主链路）：yudao-gateway → openapi → adapter →（上游数据源）；openapi → billing（扣费，OpenFeign 同步 + 幂等）。流水落库与通知推送经 yudao MQ 抽象层走 **Kafka** 异步。

### 12.3 关键技术决策

1. **主调用链不引入分布式事务**：扣费由 billing-service 本地事务完成；调用最终失败场景走自动冲正（补偿模式）；Seata 仅用于低频后台跨服务账务操作（可选）。
2. **配置快照下发**：yudao-module-route 发布配置版本 → Nacos/Redis Pub-Sub 通知 → openapi/adapter 本地缓存热更新，调用链路零数据库访问。
3. **限流分层**：网关层 Sentinel（机构/账号维度）+ openapi 内嵌并发信号量（产品维度）+ adapter 上游配额控制。
4. **虚拟线程**：openapi/adapter 为 IO 密集服务，Java 21 虚拟线程以同步编码获得高并发吞吐，避免响应式编程复杂度。
5. **雪花 ID 自研**：节点 ID 由 Nacos/DB 租约分配，时钟回拨保护见 §4.1.5，无需引入 Leaf 等外部组件。
6. **基座复用边界**：yudao-cloud 的系统管理、操作日志、代码生成直接复用（本项目按单租户模式关闭多租户）；数据面/控制面服务只依赖其 framework starter（web/mybatis/redis/mq），商城、CRM、工作流等无关业务模块不启用。

### 12.4 前端架构

- **yudao-ui-admin-vue3**：Vue 3 + TypeScript + Vite + Pinia + Element Plus，随基座框架自带。
- 直接复用：RBAC 动态菜单/按钮权限、字典组件、CRUD 列表/表单脚手架（配合代码生成器生成各管理模块基础页面）、登录与安全。
- ECharts 运营看板与监控图表。
- **Schema 化动态表单引擎**：数据源适配映射、计费模板阶梯区间、路由条件等高度动态的配置界面由 JSON Schema 驱动渲染，降低表单维护成本。
- 路由模拟器、联调测试台、流水全链路视图为重交互页面，独立组件模块开发。

### 12.5 部署拓扑

- Kubernetes 部署（一期资源有限可 Docker Compose 双节点起步）；数据面服务（gateway/openapi/adapter/flow）无状态水平扩展。
- 中间件：MySQL 主从、Redis 哨兵/集群、Kafka 集群、Nacos 集群。
- 可观测：SkyWalking（分布式链路）、Prometheus + Grafana（指标看板）、告警对接邮件/IM。

## 13. 风险与开放问题

| # | 风险/开放问题 | 应对 |
|---|---|---|
| 1 | 供应商应答码梳理工作量大，未映射码影响计费准确性 | 未映射码统一归 3001 + 告警补录清单；上线前批量导入存量映射 |
| 2 | 动态选源策略权重初值难定 | 一期只采指标不决策；二期灰度（先单产品试点，最小流量保底） |
| 3 | 阶梯计费口径（滑档 vs 返算）需与财务确认 | 本 PRD 定为滑档；如财务要求返算，计费引擎预留口径开关 |
| 4 | 个人征信合规（授权书、报文留存期）依赖法务口径 | 授权书编号字段与留存期均做成可配置 |

## 14. 核心结论

新 API 系统按**三平面架构**（管理面配置运营 / 控制面策略决策 / 数据面高并发执行）建设"数据源接入、产品封装、机构授权、路由切换、计费账务、流水审计、统计运营"一体化平台。

一期交付运营闭环：数据源可接入、产品可配置、机构可开通、路由可切换、调用可追踪、费用可计算、账单可统计、问题可排查；并从一期开始积累数据源指标，二期上线按价格/质量/时延/配额自动决策的动态路由引擎与异步/批量/开放门户能力。
