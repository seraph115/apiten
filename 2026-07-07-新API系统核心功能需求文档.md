# 新 API 系统核心功能需求文档

编写日期：2026-07-07

## 1. 背景与目标

现有旧 API 体系由 CISP 与 EDH 两类系统共同支撑。CISP 主要承载数据产品封装、机构端查询、开放接口、运营管理、图谱报告、账单成本和统计分析；EDH 主要承载外部数据源接入、数据源统一适配、字段映射、结果码归一、Dubbo 服务化输出和采集链路监控。

新 API 系统需要在保留旧系统成熟业务能力的基础上，建设一套面向征信数据服务的新一代 API 管理与运营平台，实现数据源接入配置化、产品配置化、机构开通标准化、路由切换可管控、计费账务可追溯、调用流水可审计、统计报表可运营。

本需求文档结合以下材料整理：

- 当前 oldapi 项目功能盘点与代码结构。
- `ectcispserver` 中产品、机构、路由、账单、成本、接口、流水、监控等后台模块。
- `edh-webprovider` 中数据源适配、统一 Provider、公共采集能力和监控治理能力。
- `征信系统调研` 目录下的系统管理、数据源管理、产品信息、机构管理、账户管理、路由管理、流水管理、参数管理、统计报表等界面截图。

## 2. 系统定位

新 API 系统定位为“征信数据服务统一运营平台”，对内管理数据源、产品、机构、路由、计费、流水、报表和参数，对外提供标准 API 服务、统一认证鉴权、统一响应结构、统一错误码和调用审计。

系统应覆盖三类角色：

- 运营管理人员：维护数据源、产品、机构、计费、路由、参数和报表。
- 技术接入人员：维护数据源接口、应答码、API 文档、联调配置和调用排查。
- 机构客户及其账号：通过授权产品调用 API，查看账户、账单、流水和调用统计。

## 3. 建设原则

- 配置化优先：数据源、产品、机构授权、路由、切换链、计费模板、限流规则均应后台可配置。
- 标准化输出：统一 API 请求规范、响应结构、错误码、流水编号、产品编码、机构编码和数据源编码。
- 可追溯运营：每一次 API 调用都应形成机构流水、数据源流水、计费流水和必要的日志链路。
- 可控降级：支持多数据源路由、分流、切换链、失败切换和规则启停。
- 账务闭环：从产品成本、机构计费、账户充值、账务调整到账单统计形成闭环。
- 迁移友好：旧系统中的核心产品、数据源、机构、路由、账单和流水数据应具备迁移映射方案。

## 4. 系统边界

### 4.1 需要建设的系统能力

- 新 API 后台管理系统。
- 对外 API 网关与服务入口。
- 数据源接入与适配管理。
- 产品与组合产品管理。
- 机构、账号、授权和计费管理。
- 路由、分流、切换规则和切换链管理。
- 账户、充值、财务流水和账务调整。
- 机构流水、数据源流水、邮件发送流水。
- 并发量、调用量等参数控制。
- 日账单、月账单、成本统计、月成本等统计报表。
- API 调用监控、异常排查、审计日志和运营看板。

### 4.2 可作为后续阶段建设的能力

- 图谱分析、股权穿透、最终受益人等复杂可视化产品。
- PDF/Excel 报告生成与批量查询任务。
- 开放平台门户、示例代码、在线调试和 API 文档站。
- 数据质量看板、供应商 SLA 管理和自动化对账。

## 6. 核心功能需求

### 6.1 系统管理

系统管理提供后台运行所需的基础权限和安全能力。

功能需求：

- 支持后台用户登录、退出、密码修改、密码重置和登录状态管理。
- 支持用户、角色、菜单、按钮级权限配置。
- 支持按角色分配菜单和功能权限，控制数据源、产品、机构、账务等敏感功能访问范围。
- 支持操作日志记录，记录新增、修改、删除、启停、导入、导出、审核等关键操作。
- 支持登录日志、安全审计、异常登录提示和账号启停。
- 支持系统参数、字典项、枚举值的统一维护。

验收要点：

- 不同角色登录后只能访问授权菜单。
- 关键配置变更可以追溯到操作人、操作时间、变更前值和变更后值。
- 后台所有管理列表具备分页、条件查询、详情、新增、修改、启停能力。

### 6.2 数据源管理

数据源管理用于配置外部供应商、内部数据能力及其接口，替代旧 EDH 中大量硬编码和分散配置。

#### 6.2.1 数据源信息

功能需求：

- 支持维护数据源编码、数据源名称、供应商名称、数据源类型、联系人、状态、服务地址、认证方式、备注。
- 支持配置数据源启停、环境标识、超时时间、最大并发、重试次数和是否参与路由。
- 支持维护 HTTP、HTTPS、数据库、文件、RPC 等不同接入类型的扩展参数。
- 支持查看数据源下挂接口数量、调用量、成功率、失败率和最近调用时间。

#### 6.2.2 数据源接口

功能需求：

- 支持维护接口编码、接口名称、URI、请求方式、请求参数、响应参数、接口版本、接口状态。
- 支持配置接口认证参数、报文格式、签名方式、加密方式、超时、重试和缓存策略。
- 支持数据源接口与产品功能建立映射关系。
- 支持接口新增、修改、查询、详情和启停。
- 支持接口文档、请求示例、响应示例和字段说明维护。

#### 6.2.3 数据源应答码

功能需求：

- 支持维护供应商原始应答码、应答描述、是否成功、是否计费、是否可重试、是否触发切换。
- 支持将供应商应答码映射为平台统一错误码。
- 支持按数据源、接口、产品维度查询应答码。
- 支持应答码批量导入和导出。

#### 6.2.4 分流配置

功能需求：

- 支持按产品、机构、接口、区域、请求参数、权重比例配置数据源分流。
- 支持单数据源、多个数据源权重分流和固定优先级分流。
- 支持分流配置启停、生效时间、失效时间和版本记录。
- 支持分流命中结果在流水中记录，便于排查请求进入哪个数据源。

#### 6.2.5 切换规则配置

功能需求：

- 支持按错误码、超时、无数据、调用异常、成功率阈值配置切换规则。
- 支持配置是否继续切换、切换目标、最大切换次数、切换后是否计费。
- 支持配置规则优先级，避免多个规则冲突。
- 支持规则试运行和启停。

#### 6.2.6 切换链配置

功能需求：

- 支持将多个数据源接口组织成切换链。
- 支持配置链路顺序、切换条件、终止条件、兜底数据源和返回合并策略。
- 支持链路详情查看，展示每个节点的数据源、接口、应答码规则和计费策略。
- 支持切换链版本管理，历史请求可追溯到当时命中的链路版本。

### 6.3 产品信息管理

产品信息管理负责将底层数据能力封装为对外可售卖、可授权、可计费、可路由的产品。

#### 6.3.1 产品信息管理

功能需求：

- 支持维护产品编码、产品名称、产品类型、认证类型、产品状态、产品版本、产品说明。
- 支持产品按企业、个人、核验、司法、经营风险、知识产权、报告、组合产品等类型分类。
- 支持配置产品入参、出参、示例报文、错误码、是否缓存、是否支持异步。
- 支持产品上下架、启停、版本升级和历史版本查看。

#### 6.3.2 产品功能

功能需求：

- 支持将一个产品拆分为多个产品功能或模块，例如工商基本信息、股东、主要人员、司法风险、经营异常。
- 支持维护功能编码、功能名称、所属产品、排序、是否必选、是否计费。
- 支持功能与数据源接口建立映射关系。
- 支持功能级启停，满足组合产品按模块配置。

#### 6.3.3 组合产品

功能需求：

- 支持多个产品或产品功能组合成一个对外产品。
- 支持组合产品配置执行顺序、并行/串行策略、结果合并策略、失败处理策略。
- 支持组合产品成本汇总和机构售卖价格配置。
- 支持组合产品详情查看，展示组成产品、数据源、计费规则和路由规则。

#### 6.3.4 产品数据源配置

功能需求：

- 支持产品与一个或多个数据源接口绑定。
- 支持配置产品默认数据源、备用数据源、分流规则、切换链。
- 支持按机构覆盖产品默认数据源配置。
- 支持展示产品到数据源的完整调用路径。

#### 6.3.5 产品成本管理

功能需求：

- 支持维护产品维度、数据源维度、接口维度的成本价格。
- 支持按成功、无数据、失败、超时、特定应答码配置不同成本规则。
- 支持成本生效时间、失效时间、审批状态和修改记录。
- 支持产品成本修改记录查询。

### 6.4 机构管理

机构管理负责客户资料、账号、产品授权、计费模板和用户映射。

#### 6.4.1 机构信息

功能需求：

- 支持维护机构编码、机构名称、统一社会信用代码、联系人、联系方式、机构状态、业务归属。
- 支持机构详情查看，展示账户、产品、账号、计费模板和调用统计。
- 支持机构启停，停用后应阻断 API 调用并记录原因。
- 支持机构资料变更记录。

#### 6.4.2 机构账号

功能需求：

- 支持为机构创建一个或多个 API 调用账号。
- 支持维护账号、AK/SK、密钥状态、IP 白名单、回调地址、签名算法、有效期。
- 支持密钥生成、重置、停用和历史密钥作废。
- 支持账号级调用权限和并发/调用量限制。

#### 6.4.3 机构产品管理

功能需求：

- 支持为机构开通产品或组合产品。
- 支持配置机构产品状态、有效期、单价、计费模板、调用限额、并发限制。
- 支持查看机构已开通产品详情和历史变更记录。
- 支持机构产品修改记录查询，记录开通、停用、调价、换模板等动作。

#### 6.4.4 计费模板

功能需求：

- 支持维护计费模板名称、收费模式、指定单价、最低消费、阶梯计费、免费额度、结算周期。
- 支持按产品、机构、应答码、调用结果配置计费规则。
- 支持模板详情查看，展示收费模式、阶梯区间、适用产品和适用机构。
- 支持模板复制、新增、修改、启停和版本记录。

#### 6.4.5 计费应答码

功能需求：

- 支持配置哪些平台应答码参与计费，哪些不计费。
- 支持按机构、产品、计费模板覆盖默认计费应答码。
- 支持配置成功计费、无数据计费、失败不计费、特定错误码计费等规则。

#### 6.4.6 机构用户映射与机构用户管理

功能需求：

- 支持机构用户、后台用户、业务人员之间的映射关系。
- 支持机构用户新增、修改、启停、查询和详情。
- 支持一个机构多个业务用户，满足不同客户人员查看调用、账单、流水。

#### 6.4.7 合并对账

功能需求：

- 支持按机构、产品、账期、账户维度合并对账。
- 支持机构调用流水、计费流水、财务流水之间的差异核对。
- 支持对账结果查询、导出和差异标记。

### 6.5 账户管理

账户管理负责机构余额、充值、流水和账务调整。

#### 6.5.1 计费账户

功能需求：

- 支持为机构创建计费账户，维护账户编号、账户类型、余额、冻结金额、信用额度、状态。
- 支持查询账户余额、累计充值、累计消费、最近消费时间。
- 支持账户启停、冻结、解冻和余额预警。

#### 6.5.2 账户充值

功能需求：

- 支持后台为机构账户充值。
- 支持维护充值金额、充值方式、充值凭证、经办人、审批状态、备注。
- 支持充值成功后生成财务流水，并更新账户余额。
- 支持充值记录查询和导出。

#### 6.5.3 财务流水

功能需求：

- 支持查询充值、消费、退款、调整、冻结、解冻等财务流水。
- 支持按机构、账户、流水类型、时间、产品、订单号查询。
- 支持流水详情展示关联的调用流水、账单和操作记录。

#### 6.5.4 账务调整

功能需求：

- 支持运营人员发起余额调增、调减、退款、冲正。
- 支持调整原因、凭证、审批状态和调整前后余额记录。
- 支持调整完成后生成财务流水和审计日志。

### 6.6 路由管理

路由管理是新 API 系统的核心运行控制能力，负责把机构产品请求映射到具体数据源接口。

功能需求：

- 支持通用路由配置，维护路由编码、路由名称、产品、机构、数据源、切换链、状态。
- 支持按产品默认路由、机构专属路由、机构产品路由配置。
- 支持路由目标为单数据源、多个数据源、切换链。
- 支持路由优先级：机构产品路由优先于机构路由，机构路由优先于产品默认路由。
- 支持按请求参数、地区、产品功能、账号、时间段选择路由。
- 支持路由启停、生效时间、失效时间和变更记录。
- 支持路由命中结果写入机构流水和数据源流水。

验收要点：

- 同一产品不同机构可命中不同数据源或切换链。
- 路由变更后新请求按新配置执行，历史流水保留旧路由版本信息。
- 数据源异常时可按切换规则进入备用数据源。

### 6.7 流水管理

流水管理用于支持客服排障、账务核对、供应商对账和运营分析。

#### 6.7.1 机构流水

功能需求：

- 记录机构每一次 API 调用。
- 记录流水号、机构、账号、产品、产品功能、请求时间、响应时间、耗时、请求状态、平台应答码、是否计费、计费金额。
- 记录命中路由、切换链、请求 IP、签名结果、请求摘要、响应摘要。
- 支持按机构、产品、应答码、时间、流水号查询和导出。

#### 6.7.2 数据源流水

功能需求：

- 记录每一次对外部数据源的真实调用。
- 记录数据源、接口、供应商请求号、平台流水号、请求时间、响应时间、耗时、原始应答码、平台应答码、是否切换、是否计费。
- 支持一个机构流水关联多条数据源流水。
- 支持查看切换链每个节点的调用结果。

#### 6.7.3 邮件发送流水

功能需求：

- 支持记录账单、报表、告警、通知类邮件发送流水。
- 支持查询发送对象、发送时间、发送状态、失败原因和重发记录。

### 6.8 参数管理

参数管理负责 API 调用风险控制和资源保护。

#### 6.8.1 机构并发量控制

功能需求：

- 支持按机构、账号、产品配置最大并发数。
- 支持超过并发阈值时拒绝、排队或降级。
- 支持并发控制实时生效。
- 支持查看当前并发、历史峰值和触发记录。

#### 6.8.2 机构调用量控制

功能需求：

- 支持按日、月、账期配置机构调用量上限。
- 支持按产品、账号、机构配置不同限额。
- 支持达到阈值时预警，超过阈值时阻断或进入审批。
- 支持调用量控制记录查询。

### 6.9 统计报表

统计报表用于支撑运营管理、账务结算和成本分析。

功能需求：

- 支持日账单，按机构、产品、应答码、调用量、计费量、金额汇总。
- 支持月账单，按账期生成机构月度消费汇总和明细。
- 支持成本统计，按数据源、接口、产品、时间统计供应商成本。
- 支持月成本，按数据源、产品、机构维度汇总月度成本。
- 支持收入、成本、毛利、调用量、成功率、无数据率、失败率等指标统计。
- 支持报表查询、导出、邮件发送和生成流水。

### 6.10 对外 API 服务

新系统需要提供统一对外 API 服务入口，承接旧 CISP 中企业、个人、核验、组合产品等接口能力。

功能需求：

- 支持统一 API 地址规范，按产品编码或服务路径调用。
- 支持 AK/SK、签名、时间戳、随机数、IP 白名单等认证鉴权。
- 支持同步查询、异步申请、异步结果查询。
- 支持统一请求参数校验、必填校验、格式校验、枚举校验和长度校验。
- 支持统一响应结构，包含流水号、产品编码、应答码、应答描述、数据体、耗时。
- 支持统一错误码，包括认证失败、无权限、产品停用、余额不足、限流、参数错误、数据源异常、无数据、系统异常。
- 支持请求幂等控制，避免重复计费。
- 支持接口版本化，允许同一产品多版本并存。
- 支持敏感字段脱敏、报文加密、日志脱敏。

### 6.11 数据源适配服务

数据源适配服务承接旧 EDH 的核心能力，但应尽量平台化、配置化。

功能需求：

- 支持按数据源接口配置请求模板、参数映射、响应字段映射。
- 支持 HTTP/HTTPS 接口调用，预留 RPC、数据库、文件类接入扩展。
- 支持供应商认证、签名、加密、证书、代理、超时、重试配置。
- 支持原始响应解析、字段标准化、结果码映射。
- 支持供应商异常、网络异常、超时、限流、无数据等状态归一。
- 支持采集日志、调用链、供应商请求号和原始报文留痕。
- 支持数据源接口级健康检查和可用性统计。

### 6.12 监控与运维

功能需求：

- 支持 API 调用量、成功率、失败率、平均耗时、P95/P99 耗时监控。
- 支持数据源可用率、超时率、错误码分布、切换次数监控。
- 支持机构调用异常、余额不足、限额触发、数据源异常告警。
- 支持按流水号快速定位机构请求、路由命中、数据源调用和计费结果。
- 支持配置缓存刷新，保证路由、产品、机构、计费模板变更后快速生效。
- 支持关键任务调度，例如日账单、月账单、成本统计、流水归档。

## 7. 关键业务流程

### 7.1 数据源接入流程

1. 新增数据源信息。
2. 新增数据源接口，配置 URI、请求方式、认证、参数、响应字段。
3. 配置数据源应答码并映射为平台统一应答码。
4. 配置接口成本和是否参与计费。
5. 将数据源接口绑定到产品功能。
6. 配置路由、分流或切换链。
7. 联调验证通过后启用。

### 7.2 产品上架流程

1. 新增产品信息，维护产品编码、名称、类型、版本。
2. 新增产品功能，配置入参、出参、是否计费。
3. 绑定数据源接口或组合已有产品。
4. 配置产品默认路由、成本和统一错误码。
5. 配置 API 文档、示例报文和版本说明。
6. 产品启用并允许机构授权。

### 7.3 机构开通流程

1. 新增机构信息。
2. 创建机构账号并生成 AK/SK。
3. 配置 IP 白名单、签名方式和有效期。
4. 开通机构产品。
5. 绑定计费模板、调用量控制和并发控制。
6. 创建计费账户并完成充值或授信。
7. 联调通过后正式启用。

### 7.4 API 调用流程

1. 机构账号发起 API 请求。
2. 系统完成签名、IP、账号、机构、产品权限校验。
3. 系统校验余额、调用量、并发量和产品状态。
4. 系统根据产品、机构、账号、参数命中路由。
5. 系统调用数据源或切换链。
6. 系统完成结果码映射、响应组装和错误处理。
7. 系统生成机构流水、数据源流水、计费流水。
8. 系统返回统一响应给调用方。

### 7.5 计费与对账流程

1. API 调用结束后根据产品、机构、计费模板和应答码判断是否计费。
2. 计费成功后扣减账户余额或记录后付费账单。
3. 生成机构流水、计费明细和财务流水。
4. 日终生成日账单，月终生成月账单。
5. 运营人员按机构、产品、数据源进行收入成本核对。
6. 发现差异后通过合并对账和账务调整处理。

## 8. 数据对象需求

新 API 系统至少需要维护以下核心数据对象：

- 用户、角色、菜单、权限、操作日志。
- 数据源、数据源接口、数据源应答码、接口字段、接口认证配置。
- 产品、产品功能、组合产品、产品数据源关系、产品成本。
- 机构、机构账号、机构产品、机构用户、机构用户映射。
- 计费模板、计费应答码、计费账户、充值记录、财务流水、账务调整。
- 路由配置、分流配置、切换规则、切换链、路由版本。
- 机构流水、数据源流水、计费流水、邮件发送流水。
- 并发控制、调用量控制、阈值配置、预警记录。
- 日账单、月账单、成本统计、对账记录。

## 9. 非功能需求

### 9.1 性能

- API 服务应支持高并发调用，核心查询链路应具备连接池、缓存、限流和超时控制。
- 后台列表查询应支持分页、索引优化和条件组合查询。
- 流水和报表应考虑分表、归档或冷热数据分离。

### 9.2 可用性

- 数据源异常时支持超时、重试、切换链和兜底响应。
- 核心配置应支持缓存与快速刷新，避免每次调用访问数据库。
- 账单、统计、报表等任务应支持失败重跑和幂等处理。

### 9.3 安全

- API 调用必须完成账号、签名、IP、产品权限校验。
- 密钥、身份证号、手机号、银行卡号等敏感数据应加密存储或脱敏展示。
- 日志中不得明文输出完整密钥、证件号、手机号和完整请求报文中的敏感字段。

### 9.4 审计

- 产品、机构、计费、路由、数据源等关键配置变更必须有审计记录。
- 账务调整、充值、退款、冲正必须记录操作人、审批状态和凭证。
- API 调用必须可通过流水号追溯到机构、产品、路由、数据源、计费结果。

### 9.5 可扩展性

- 新增数据源不应影响既有产品。
- 新增产品应优先通过配置组合已有数据源接口实现。
- 同一产品应支持多版本共存，避免升级影响存量机构。

## 10. 一期建设范围建议

一期建议优先建设能够支撑 API 正式运营闭环的能力。

### 10.1 一期必须建设

- 系统管理：用户、角色、菜单、权限、操作日志。
- 数据源管理：数据源信息、数据源接口、数据源应答码。
- 产品信息：产品、产品功能、产品数据源、产品成本。
- 机构管理：机构信息、机构账号、机构产品、计费模板、计费应答码。
- 账户管理：计费账户、账户充值、财务流水、账务调整。
- 路由管理：通用路由、分流配置、切换规则、切换链。
- 流水管理：机构流水、数据源流水。
- 参数管理：机构并发量控制、机构调用量控制。
- 统计报表：日账单、月账单、成本统计、月成本。
- 对外 API：认证鉴权、同步查询、统一响应、错误码、流水、计费。

### 10.2 一期可视资源情况纳入

- 邮件发送流水。
- 合并对账。
- 机构产品修改记录。
- 产品成本修改记录。
- API 文档与示例报文维护。
- 基础监控看板和告警。

### 10.3 二期建议建设

- 异步查询、批量查询和批量结果下载。
- 开放平台门户、在线调试和 SDK 示例。
- 图谱分析、股权穿透、最终受益人、报告生成。
- 供应商 SLA 分析、自动对账、数据质量看板。
- 数据源低代码适配，减少新增数据源时的开发工作量。

## 11. 验收标准

- 能通过后台完成数据源、接口、应答码、产品、机构、账号、路由、计费模板的完整配置。
- 机构账号可使用 AK/SK 调用已授权产品，未授权产品调用失败且返回统一错误码。
- API 调用能按路由配置命中指定数据源，数据源异常时能按切换规则进入备用链路。
- 每次调用均生成机构流水和数据源流水，并能通过流水号查询完整链路。
- 成功调用能按计费模板生成计费结果，账户余额、财务流水、日账单、月账单数据一致。
- 机构并发量、调用量达到阈值时能够拦截或预警。
- 产品、机构、计费、路由、数据源关键配置变更有操作审计。
- 报表可按机构、产品、数据源、时间维度查询和导出。

## 12. 与旧系统能力的迁移映射

| 旧系统能力 | 新系统承接方式 |
| --- | --- |
| CISP `ApiProduct` 产品封装 | 产品信息、产品功能、组合产品、对外 API 服务 |
| CISP `DataSrcService` 数据源调用 | 产品数据源配置、路由管理、数据源适配服务 |
| CISP `Module` 结果加工 | 产品功能、响应字段映射、组合产品结果组装 |
| CISP 机构配置与授权 | 机构信息、机构账号、机构产品管理 |
| CISP 路由配置 | 通用路由、分流配置、切换规则、切换链 |
| CISP 成本、账单、报表 | 产品成本、计费模板、账单、成本统计 |
| CISP 流水、HBase/OTS 流水处理 | 机构流水、数据源流水、计费流水、流水归档 |
| EDH `gather-source-*` 数据源模块 | 数据源接口配置与适配服务 |
| EDH 统一 Provider | 新 API 数据源适配服务与统一调用入口 |
| EDH 结果码与校验配置 | 数据源应答码、平台统一错误码、参数校验 |
| CAT/SkyWalking/监控能力 | 监控运维、调用链、异常告警、指标看板 |

## 13. 核心结论

新 API 系统不是单纯重做一个接口服务，而是要建设一套“数据源接入、产品封装、机构授权、路由切换、计费账务、流水审计、统计运营”一体化平台。

一期建设的核心闭环应是：

数据源可接入、产品可配置、机构可开通、路由可切换、调用可追踪、费用可计算、账单可统计、问题可排查。
